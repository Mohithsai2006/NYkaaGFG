## 🔴 The Problem

In today's digital marketing landscape, companies like **Nykaa** generate massive volumes of campaign data daily — spanning impressions, clicks, leads, conversions, revenue, ROI, acquisition costs, and more across dozens of channels, languages, and audience segments.

### The Current Reality:

| Challenge | Impact |
|-----------|--------|
| **Data is locked in spreadsheets** | Marketing executives can't quickly explore or visualize campaign performance |
| **SQL/technical skills required** | Business teams depend on data analysts for every simple question, creating bottlenecks |
| **Slow report turnaround** | Getting answers to "Which campaign type has the best ROI?" takes hours or days |
| **Static dashboards** | Pre-built BI tools show fixed views — they can't answer ad-hoc, evolving questions |
| **No conversational flow** | Users can't ask follow-up questions like "Now filter that to Hindi campaigns only" |
| **Tool complexity** | Tools like Tableau, Power BI require training, licenses, and setup |

### The Core Pain Point:

> **Non-technical business stakeholders (CXOs, marketing managers, campaign leads) cannot independently explore their own data.** They must rely on technical teams to write SQL queries, build charts, and deliver insights — creating delays,  miscommunication, and missed opportunities.

---

## 💡 Why It Matters

In a competitive market, **speed of insight = competitive advantage**. When a marketing head spots a declining channel ROI, they need to act *today*, not wait three days for a report. The gap between "I have a question" and "I have an answer" directly impacts business decisions worth lakhs of rupees.

**What's needed:**
- A tool that any business user can use — **no SQL, no code**
- **Instant responses** with interactive visualizations
- **Conversational interface** that supports natural follow-ups
- **Flexibility** to upload new datasets on the fly
- **Intelligence** to pick the right chart type automatically

## ✅ Our Solution

**Nykaa BI** is a **Conversational AI-Powered Business Intelligence Dashboard** that lets any user — regardless of technical background — ask business questions in **plain English** and receive **instant, interactive charts** generated by Google Gemini AI.

### What We Built:

A full-stack web application with three core components:

1. **AI Agent (Brain)** — Powered by Google Gemini 2.5 Flash, it understands natural language questions, translates them into optimized SQL queries, selects the best chart type, and provides key insights.

2. **Data Engine (Backend)** — A Node.js/Express server that loads CSV marketing data into an in-memory SQL database (AlaSQL), executes AI-generated queries safely, and returns structured results.

3. **Interactive Dashboard (Frontend)** — A stunning Next.js application with a conversational chat panel, real-time chart rendering (6 chart types), and a modern dark glassmorphism UI.

## ⚙️ How It Works

### The Pipeline: Question → Answer in 3 Seconds

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  User asks   │────▶│  Node.js     │────▶│  Gemini AI   │────▶│  AlaSQL      │────▶│  Recharts    │
│  a question  │     │  Backend     │     │  Agent       │     │  Database    │     │  Dashboard   │
│  in English  │     │  receives it │     │  NL → SQL +  │     │  executes    │     │  renders     │
│              │     │              │     │  chart type  │     │  the query   │     │  the chart   │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
```

### Step-by-Step:

1. **User types**: *"Show revenue by campaign type"*
2. **Backend** sends the question + database schema to Gemini AI
3. **Gemini** returns a structured JSON response:
   ```json
   {
     "sql": "SELECT Campaign_Type, SUM(Revenue) as Total_Revenue FROM campaigns GROUP BY Campaign_Type ORDER BY Total_Revenue DESC LIMIT 10",
     "chart_type": "bar",
     "x_axis": "Campaign_Type",
     "y_axis": "Total_Revenue",
     "title": "Revenue by Campaign Type",
     "insight": "Social Media campaigns generate the highest revenue."
   }
   ```
4. **Backend** validates the SQL (blocks any destructive queries), executes it against the in-memory database, and returns the results
5. **Frontend** renders an interactive bar chart with tooltips, legends, and the AI-generated insight
6. **User can follow up**: *"Now show that as a pie chart"* — the AI remembers the conversation context

## ✨ Key Features

### 🤖 AI-Powered Intelligence
- **Natural Language to SQL** — Ask questions in plain English; Gemini converts them to optimized SQL
- **Smart Chart Selection** — AI automatically picks the best visualization (bar, line, pie, area, scatter, or table)
- **Key Insights** — Every chart comes with an AI-generated one-line insight
- **Conversation Memory** — Ask follow-up questions; the AI remembers previous context

### 📊 6 Interactive Chart Types
| Chart Type | Best For | Example Query |
|-----------|----------|---------------|
| **Bar** | Comparing categories | "Revenue by campaign type" |
| **Line** | Trends over time | "Monthly ROI trend" |
| **Pie** | Parts of a whole | "Revenue share by language" |
| **Area** | Cumulative trends | "Cumulative clicks over time" |
| **Scatter** | Correlations | "ROI vs Acquisition Cost" |
| **Table** | Detailed data | "Top 10 campaigns by revenue" |

### 🛡️ Safety & Reliability
- **SQL Injection Guard** — Only SELECT statements are allowed; INSERT, DELETE, DROP are blocked
- **Hallucination Prevention** — AI gracefully declines when data doesn't support the question
- **Rate Limit Handling** — Automatic retry with exponential backoff on API throttling
- **Model Fallback Chain** — Falls back through Gemini 2.5 Flash → 2.0 Flash → 1.5 Flash

### 📎 Data Flexibility
- **Default Dataset** — Ships with 55,000+ Nykaa digital marketing campaign records
- **CSV Upload** — Upload any CSV file and start querying it instantly
- **Auto Schema Detection** — Automatically detects column types (numeric, string, date)

### 🎨 Premium UI/UX
- **Dark Glassmorphism Design** — Frosted glass panels, subtle gradients, micro-animations
- **Collapsible Chat Panel** — Toggle sidebar for full-screen dashboard view
- **Responsive Layout** — Works on desktop and tablet screens
- **Suggestion Chips** — Pre-built query suggestions to get started quickly
- **Real-time Status Indicators** — "AI Ready" badge, typing indicators, connection status

## 🚀 Getting Started

### Prerequisites

- **Node.js** v18+ ([download](https://nodejs.org/))
- **npm** (comes with Node.js)
- **Google Gemini API Key** (free, instructions below)

### Step 1: Clone the Repository

```bash
git clone https://github.com/Mohithsai2006/NYkaaGFG.git
cd NYkaaGFG
```

### Step 2: Get a Gemini API Key (Free)

1. Go to [Google AI Studio](https://aistudio.google.com/app/apikey)
2. Sign in with your Google account
3. Click **"Create API Key"**
4. Copy the key

### Step 3: Configure the Backend

```bash
cd nodebackend
npm install
```

Create a `.env` file in the `nodebackend/` folder:

```env
GEMINI_API_KEY=your_api_key_here
PORT=8000
```

### Step 4: Configure the Frontend

```bash
cd ../frontend
npm install
```

Create a `.env.local` file in the `frontend/` folder:

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

### Step 5: Start the Application

**Terminal 1 — Backend:**
```bash
cd nodebackend
npm run dev
```

You should see:
```
[DB] Loaded 55557 rows into 'campaigns'.
[API] ✅ Gemini API key loaded successfully (AIzaSyDR...gW_Q)
[API] Nykaa BI Backend running on http://localhost:8000
```

**Terminal 2 — Frontend:**
```bash
cd frontend
npm run dev
```

You should see:
```
✓ Ready in 2s
- Local: http://localhost:3000
```

### Step 6: Open the App

Navigate to **http://localhost:3000** in your browser 🎉

## 📊 Dataset Description

The default dataset (`Nykaa Digital Marketing.csv`) contains **55,557 campaign records** with the following columns:

| Column | Type | Description |
|--------|------|-------------|
| Campaign_ID | String | Unique campaign identifier (e.g., NY-CMP-1000) |

| Impressions | Number | Number of ad impressions |
| Clicks | Number | Number of clicks received |
| Leads | Number | Number of leads generated |
| Conversions | Number | Number of successful conversions |

| Customer_Segment | String | Customer segmentation category |
| Date | String | Campaign date |
